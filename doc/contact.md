# 约定

## 代码命名规范

- 所有目录以及资源和代码文件都采用小写命名，单词间用横线隔开
- 所有类名、静态成员采用大驼峰命名法（PascalCase）
- 所有函数名和变量名采用小驼峰命名法（camelCase）
- 私有成员使用双下划线前缀（\__），保护成员使用单下划线前缀（_）
- 所有常量名采用全大写命名法（UPPER_SNAKE_CASE）
- 所有枚举名和枚举成员名都采用大驼峰命名法（PascalCase）
- 所有接口名采用大驼峰命名法（PascalCase）

## 资源命名规范

- 三段式命名：类别-模块-描述
- 类别 _（必填）_
  - **ani** 动画
  - **atl** 图集
  - **aud** 音频
  - **img** 图片
  - **cfg** 配置
  - **txt** 文本 (JSON、TXT、XML 等)
  - **fnt** 字体
  - **bin** 二进制
  - **tmx** 瓦片地图
  - **map** 地图
  - **pfb** 预制体
  - **ptc** 粒子
  - **mat** 材质
  - **vid** 视频
- 模块 _（可选）_
- 描述 _（必填）_
